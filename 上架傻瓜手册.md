## 第 1 步：建数据库表（Supabase 网页，复制粘贴）
1. 打开 [supabase.com](https://supabase.com) → 你的项目 → 左侧 **SQL Editor** → **New query**。
2. 把仓库根目录 **`数据库一键部署.sql`** 的全部内容复制进去 → 点 **Run**。（这段是幂等的，重复跑也安全。）
3. **引导你自己为超级管理员**：再 New query，粘下面这段，把邮箱换成**你要登录后台用的邮箱**，Run：
   ```sql
   insert into public.admin_users (user_id, email, display_name, role)
   select id, email, '超级管理员', 'superadmin' from auth.users where email = '你的邮箱@xxx.com'
   on conflict (user_id) do update set role = 'superadmin';
   ```
   （前提：这个邮箱先在 App 注册/登录过一次，或在 Supabase → Authentication → Users 里 Add user。）
4. **（可选）开启定时采集预警**：再 New query，粘下面两行，把 `<PROJECT_REF>` 换成你的项目 ref（`aurowjqmjofpitsmlhmg`），随机串自拟，Run：
   ```sql
   alter database postgres set app.settings.functions_url = 'https://aurowjqmjofpitsmlhmg.supabase.co/functions/v1';
   alter database postgres set app.settings.cron_secret   = '随便一串长随机字符';
   ```

## 第 2 步：在 GitHub 填密钥
仓库 → **Settings** → 左侧 **Secrets and variables** → **Actions** → 右上 **New repository secret**，按下表逐个加（名字照抄，值粘进去）。

### 必填（最小可用：预警/SOS/后台/内购能跑）
| Secret 名字 | 值从哪来 |
|---|---|
| `SUPABASE_ACCESS_TOKEN` | supabase.com → 右上头像 → **Access Tokens** → Generate new token，整串复制 |
| `APPSTORE_ISSUER_ID` | **复用**你 iOS 上传那套：和已有的 `APP_STORE_CONNECT_API_ISSUER_ID` 填一样的值 |
| `APPSTORE_KEY_ID` | 复用：和 `APP_STORE_CONNECT_API_KEY_ID` 一样 |
| `APPSTORE_PRIVATE_KEY` | 复用：和 `APP_STORE_CONNECT_API_KEY_CONTENT` 一样（那段 .p8 全文） |
| `APPSTORE_BUNDLE_ID` | 填 `com.warrescue.appname` |

> iOS 上传用的 `APP_STORE_CONNECT_API_KEY_ID / _ISSUER_ID / _KEY_CONTENT / APPLE_TEAM_ID` 你**之前应该已经配过**（不然之前传不上去）。没配就照旧补上。

### 建议填（完整功能：推送、SOS 短信）
| Secret 名字 | 值从哪来 |
|---|---|
| `APNS_KEY_ID` | Apple Developer → Certificates,IDs&Profiles → **Keys** → 新建 Key 勾 **APNs** → 记下 Key ID |
| `APNS_TEAM_ID` | 你的 Apple Team ID（开发者账号右上角） |
| `APNS_BUNDLE_ID` | `com.warrescue.appname` |
| `APNS_AUTH_KEY` | 上面新建 APNs Key 时下载的 `.p8` 文件，用记事本打开，**整段**粘进来 |
| `TWILIO_ACCOUNT_SID` | Twilio 控制台首页（AC 开头） |
| `TWILIO_FROM_NUMBER` | 你在 Twilio 买的号码（+ 开头） |
| `TWILIO_KEY_SID` / `TWILIO_KEY_SECRET` | Twilio → API keys & tokens → Create API key |

### 选填
| Secret 名字 | 说明 |
|---|---|
| `FCM_SERVICE_ACCOUNT` | 安卓推送：Firebase → 项目设置 → 服务账号 → 生成私钥(JSON 全文)。不填则安卓不推送 |
| `MEOO_PROJECT_API_KEY` | AI 预警分析密钥（有就填） |
| `CRON_SECRET` | 和第 1 步第 4 点那个随机串一致 |

> 多行的值（`.p8`、JSON）整段粘贴即可，GitHub 支持多行。

## 第 3 步：在 GitHub 点"运行"
仓库 → 顶部 **Actions** → 左侧选流程 → 右侧 **Run workflow**。按顺序：
1. **Deploy Supabase (Edge Functions + Secrets)** → Run。（部署后端 + 写入你填的密钥）
2. **Build iOS & Upload to App Store Connect** → Run（`upload_to_appstore` 保持 true）。约 15–30 分钟，成功后构建会出现在 App Store Connect / TestFlight。
3. （可选）**Build Android (APK)** → Run，跑完在该次运行页面底部下载 APK 装手机测试。

---

## 第 4 步：App Store Connect 填资料提交
到 [App Store Connect](https://appstoreconnect.apple.com) 选中你的 App：
- 选刚上传成功的**构建版本**。
- 填：截图（用仓库 `ipad-screenshots/` 的 iPad 图 + iPhone 截图）、描述、关键词、**App 隐私问卷**（照 `合规与上架材料.md` 第 3 节勾）、年龄分级、**审核备注**（照第 5 节：说明是民用预警工具 + 模拟演习可演示）。
- 隐私政策 URL：把网页版部署到公网后填（见 `部署总清单.md`），或先用一个能访问的页面。
- 点 **提交以供审核**。

## 卡住了怎么办
任何一步的报错（GitHub Actions 里红色 ❌、Supabase 报错），**把那段日志/截图发我**，我帮你看。
- 推送/短信/AI 没填密钥 → 对应功能自动跳过，不影响过审，之后补上密钥重跑第 3 步第 1 个流程即可。
- 离线地图：现在用免费 OSM，能用；规模化后在 `.env` 填 `MAP_TILE_URL` 换授权源（见 `.env.example`）。
