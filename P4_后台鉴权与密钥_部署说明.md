## 五、修苹果内购 JWS 验签（已实现：方案 B）
已把 `apple-iap` 的 StoreKit2 JWS 从“只解码不验签”改为**向 Apple App Store Server API 权威重核验**：
- `verifyJWSTransaction`：只从客户端 JWS 取 transactionId，再调 Apple `/inApps/v1/transactions/{id}` 拿权威交易数据；**核验不过一律拒绝放行**（绝不 fail-open）。
- 服务器通知 `handleAppleNotification`：同样先核验再处理，防止伪造通知篡改订阅状态。
- 生产环境查不到交易会自动回退沙盒；旧的 verifyReceipt（含收据）路径不受影响。

需要的 Secrets（App Store Connect → 用户与访问 → 集成 → App Store Server API → 生成密钥）：
```bash
supabase secrets set APPSTORE_ISSUER_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
supabase secrets set APPSTORE_KEY_ID=XXXXXXXXXX
supabase secrets set APPSTORE_BUNDLE_ID=com.your.realbundleid
supabase secrets set APPSTORE_PRIVATE_KEY="$(cat SubscriptionKey_XXXXXXXXXX.p8)"
supabase functions deploy apple-iap
```
> ⚠️ 未配置这些密钥时，JWS 路径会**拒绝所有交易**（安全优先，宁可拦错不可放过伪造）。配置后即生效。
